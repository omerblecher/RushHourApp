  <action>
    Present the user with the 4-step verification procedure below. Wait for the user to return results (cap sync output, Android Studio build result, app launch result, adb grep output). If the Gradle build fails with a manifest merger error mentioning `AD_SERVICES_CONFIG`, apply the RESEARCH Pitfall 3 fallback automatically (see step 2 fallback below), then ask the user to retry the build.

    **Step 1 — Re-run cap sync and capture output (SETUP-01):**
    Run `npx cap sync` from project root.
    Expected: Output contains `@capacitor-community/admob (8.0.0)` under "Found X Capacitor plugin for android:". No red error lines.

    **Step 2 — Open project in Android Studio and build (SETUP-03):**
    1. Open `android/` in Android Studio (File → Open → select the `android` folder).
    2. Wait for Gradle sync to complete. Expected: "Gradle sync finished" with no red errors.
    3. Click Build → Make Project (Ctrl+F9).
    4. Expected: BUILD SUCCESSFUL.

    **Step 2 fallback (apply only if build fails with `ProcessLibraryManifest` or `AD_SERVICES_CONFIG` error):**
    Edit `android/app/src/main/AndroidManifest.xml`:
    a) Add `xmlns:tools="http://schemas.android.com/tools"` attribute to the root `<manifest>` tag so it becomes:
       ```xml
       <manifest xmlns:android="http://schemas.android.com/apk/res/android"
                 xmlns:tools="http://schemas.android.com/tools">
       ```
    b) Inside `<application>`, after the AdMob App ID meta-data, add:
       ```xml
       <property
           android:name="android.adservices.AD_SERVICES_CONFIG"
           android:resource="@xml/gma_ad_services_config"
           tools:replace="android:resource" />
       ```
    c) Re-sync Gradle in Android Studio and retry Build → Make Project.
    d) Record in the summary that the fallback was applied (Phase 10 needs to know).

    **Step 3 — Launch app on emulator or device (SETUP-02):**
    1. Start an Android emulator (API 33+) or connect a device with USB debugging.
    2. Click Run (Shift+F10).
    3. Expected: App launches to puzzle selection screen. NO `IllegalStateException: The Google Mobile Ads SDK was initialized incorrectly` in Logcat.
    4. Filter Logcat by the `applicationId` from `android/app/build.gradle` and verify no red ads-SDK error lines during launch.

    **Step 4 — Verify AD_ID permission in compiled APK (SETUP-04):**
    Read `applicationId` from `android/app/build.gradle` (likely `com.otis.brooke.rushhour.puzzle` based on STATE.md notes, or `com.rushhour.puzzle` based on strings.xml). Run:
    ```
    adb shell dumpsys package <applicationId> | grep -i AD_ID
    ```
    Expected: Output contains `com.google.android.gms.permission.AD_ID` (typically with `granted=true`).
