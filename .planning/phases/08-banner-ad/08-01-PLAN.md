## Threat Model (ASVS L1)

Phase 8 introduces no new security surface. The AdMob banner renders inside a native SDK overlay; no user data is passed to the ad API; the only project-controlled value is the banner unit ID, which is sourced from `import.meta.env.VITE_ADMOB_BANNER_ID` (env-gated, established in Phase 6).

| Threat ID | Category | Component | Disposition | Mitigation |
|-----------|----------|-----------|-------------|------------|
| T-08-01 | I (Information Disclosure) | `adService.showBanner` ad unit ID | accept | ID is non-secret AdMob test ID in dev (`isTesting: import.meta.env.DEV`); production ID swap deferred to Phase 10 |
| T-08-02 | T (Tampering) | env var `VITE_ADMOB_BANNER_ID` | mitigate | Sourced from `.env.development` only (already gitignored convention from Phase 6); never user-supplied |
| T-08-03 | D (Denial of Service) | Banner load failure crashing game | mitigate | `bannerHeight` defaults to 0; no error UI; failure path produces no exception that bubbles into React render — verified by unit test |

**ASVS Controls:** V5.1 (input validation N/A — no user input); V14.2.1 (dependency on pinned plugin v8.0.0 — verified in Phase 6).
</threat_model>

```ts
// Banner ad lifecycle (Phase 8 — BANNER-01..05)
useEffect(() => {
  if (!Capacitor.isNativePlatform()) return;

  let listenerHandle: PluginListenerHandle | null = null;

  AdMob.addListener(
    BannerAdPluginEvents.SizeChanged,
    (info: AdMobBannerSize) => {
```
