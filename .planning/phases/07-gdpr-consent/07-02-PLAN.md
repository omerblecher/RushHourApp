    Step 1 — Edit `src/screens/ProfileScreen/ProfileScreen.tsx`:
    - Add import at top alongside other service imports: `import { showConsentForm } from '../../services/adService';`
    - Add a new handler function inside the component (near other handlers like `handleSignOut`):
      ```tsx
      const handlePrivacySettings = async () => {
        try {
          await showConsentForm();
        } catch (err) {
          // Per RESEARCH.md Pitfall 5: showPrivacyOptionsForm may reject when
          // privacyOptionsRequirementStatus is NOT_REQUIRED (non-EEA users).
          // Swallow gracefully — do not surface to user, log only in dev.
          if (import.meta.env.DEV) console.warn('Privacy settings form unavailable:', err);
        }
      };
      ```
    - Add a new `<section>` IMMEDIATELY AFTER the existing Sign Out section (after line 182 closing `</section>`, before the card's closing `</div>`):
      ```tsx
      {/* Privacy / consent settings — GDPR-05 entry point */}
      <section className={styles.section}>
        <button
          className={styles.privacyButton}
          onClick={handlePrivacySettings}
          aria-label="Open privacy settings"
        >
          Privacy Settings
        </button>
      </section>
      ```
    - Do NOT conditionally hide the button based on geography. Per RESEARCH.md Open Question 1, the try/catch approach is lower risk than conditional hiding. The button is always visible; non-EEA users who tap it get a silent no-op (error swallowed).
