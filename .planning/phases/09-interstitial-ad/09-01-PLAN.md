4. **Append `showInterstitialIfDue`** after `prepareInterstitial`:
```typescript
export async function showInterstitialIfDue(): Promise<void> {
  _winCount++;
  if (_winCount % 3 !== 0) return;
  await waitForConsent();
  const showAndWait = new Promise<void>((resolve) => {
    void AdMob.addListener(InterstitialAdPluginEvents.Dismissed, () => {
      void prepareInterstitial(); // reload for next trigger
      resolve();
    });
    void AdMob.showInterstitial();
  });
  const timeout = new Promise<void>((res) => setTimeout(res, AD_TIMEOUT_MS));
  await Promise.race([showAndWait, timeout]);
}
```

CRITICAL constraints:
- The counter increment MUST be the first statement (so non-3rd calls still tick the counter for the next call)
- The early-return on `_winCount % 3 !== 0` MUST happen BEFORE `await waitForConsent()` (a missed call must NOT block on consent)
- Do NOT add listener cleanup after `Promise.race` — locked design intentionally omits it (RESEARCH Pitfall 2 acknowledged)
- Do NOT persist `_winCount` to localStorage / sessionStorage / Zustand — locked design requires session-only
- Use `void AdMob.showInterstitial()` (fire-and-forget) — the Dismissed listener is what resolves the promise, NOT the showInterstitial promise
- Reload via `void prepareInterstitial()` is intentionally fire-and-forget inside the listener
