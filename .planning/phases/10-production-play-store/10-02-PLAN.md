<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| .env.production → Vite build | Real AdMob IDs baked into JS bundle at build time — IDs are non-secret (visible in APK) but must not be confused with test IDs |
| Keystore credentials → Gradle signing | KEYSTORE_PASS and KEY_PASS from environment variables — must not be committed to git |
| GitHub Pages → public internet | Privacy policy page is publicly accessible — must not contain secrets |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-05 | Information Disclosure | KEYSTORE_PASS / KEY_PASS in shell environment | mitigate | Credentials read from env vars only (not hardcoded in build.gradle); not committed; .gitignore already excludes .env files |
| T-10-06 | Tampering | .env.production TODO values accidentally shipped | mitigate | Plan 10-03 Task 1 verifies AAB versionCode before upload; adService.ts would call AdMob with "TODO" string which would fail gracefully |
| T-10-07 | Information Disclosure | Real AdMob App ID in strings.xml committed to git | accept | AdMob App IDs are designed to be public (baked into APK/AAB manifest); not a secret per Google's own documentation |
| T-10-08 | Spoofing | privacyPolicyUrl pointing to wrong URL | mitigate | User provides exact URL in resume signal; Claude sets it verbatim; Plan 10-03 checkpoint asks user to verify the UMP form in Play Console shows correct URL |
</threat_model>
