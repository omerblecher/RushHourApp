---

**Action 1 — Upload AAB to internal testing track:**

1. Open Play Console → Rush Hour Puzzle → Testing → Internal testing
2. Click "Create new release" (or "Edit release" if a draft exists)
3. Under "App bundles", click "Upload" and select:
   `android/app/build/outputs/bundle/release/app-release.aab`
4. Wait for the upload to process. Play Console will show:
   - Version code: 2
   - Version name: 1.1
5. If Play Console flags any errors (policy violations, missing declarations), read the error and report it in the resume signal
6. Add release notes (optional but recommended): e.g., "Ad monetization — banner and interstitial ads with GDPR consent"
7. Click "Review release" → "Start rollout to internal testing"

EXPECTED: Upload completes without errors. The release shows version code 2 in the track listing.

---

**Action 2 — Enable "Contains ads" in app content:**

1. In Play Console, go to: Policy → App content → Ads
   (Or: Dashboard → App content → Ads)
2. Check the box: "This app contains ads"
3. Save

EXPECTED: A small "Contains ads" badge will appear on your Play Store listing.

---

**Action 3 — Update Data Safety section:**

1. In Play Console, go to: Policy → App content → Data safety
2. Click "Edit" (or start the form if not yet filled)
3. Under "Data collected":
   - Add: **Advertising or marketing → Advertising IDs**
     - Purpose: Advertising or marketing
     - Is this data shared? Yes → with Google AdMob
     - Is processing optional? Yes (user can opt out via EEA consent dialog)
4. Review all existing declarations (Firebase Auth collects email address — confirm this was already declared from v1.0)
5. Submit the form

EXPECTED: Data safety form is saved with advertising ID declared.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Release AAB → Play Store | Signed AAB submitted to Google's servers; must be signed with the production keystore (already configured in Phase 10-02 build) |
| Play Console → public listing | Data Safety declarations are legally binding representations to users; accuracy is required |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-09 | Tampering | Uploading a debug AAB instead of release AAB | mitigate | Task 1 verifies the AAB path and versionCode before user uploads; debug AABs are not signed with the release key and would be rejected by Play Console anyway |
| T-10-10 | Information Disclosure | Play Console policy violation disclosure | accept | If Play Console flags a policy issue, the release is blocked before reaching users — disclosure is contained to the developer |
| T-10-11 | Repudiation | Incomplete Data Safety declaration (omitting advertising ID) | mitigate | Action 3 instructions explicitly list advertising ID as required; omitting it violates Play policy and risks app removal |
| T-10-12 | Denial of Service | Stale AAB from previous build (versionCode 1) uploaded | mitigate | Task 1 automated check confirms versionCode 2 in build.gradle before user proceeds to upload; Play Console would reject a lower versionCode anyway |
</threat_model>
